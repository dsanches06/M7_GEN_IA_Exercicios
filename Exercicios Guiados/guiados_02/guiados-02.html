⚙️ Parte 3 — Gemini Avançado

Exercício 8 — Thinking Mode (Planeamento de Feature Real)

Objetivo: Usar o modo de raciocínio do Gemini para planear uma feature complexa de produto com várias decisões técnicas.

Cenário real (ClickBot AI):

A equipa quer adicionar uma nova funcionalidade ao ClickBot: "AI Task Auto-Split"

Quando um utilizador escreve uma tarefa grande, por exemplo: 

"Criar sistema completo de login com OAuth, recuperação de password e 2FA"

A AI deve transformar isso em várias subtarefas automaticamente no ClickUp.

Tarefa: Criar a função generateTaskBreakdown() que usa Thinking Mode para planear como esta feature deve funcionar.

Resultado esperado:

  • Definição de regras para splitting de tarefas
  • Estrutura JSON para subtarefas
  • Regras para não over-split
  • Fluxo frontend → backend → AI
  • Casos edge (tarefas curtas / vagas)
Dica: O Thinking Mode deve ser usado quando há decisões de arquitetura ou múltiplas etapas envolvidas.
