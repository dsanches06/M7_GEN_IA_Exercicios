ClickBot Builder: Workshop Prático GenAI

Construa funcionalidades de próxima geração para o "ClickBot", o assistente inteligente da nossa plataforma de produtividade.

Streaming Database

1. Chat de Suporte com Memória Persistente

Contexto: O utilizador precisa de ajuda sobre como usar o ClickUp. A resposta deve aparecer gradualmente para melhorar a UX.

Tarefa:

  • No Express, crie uma rota GET /chat que retorna um text/event-stream.
  • Consuma o result.stream do Gemini e mostre cada chunk na consola .
  • Após o stream terminar ([DONE]), guarde a user_message e a ai_response completa na tabela chat_history.
Structured Output

2. Smart Task Parser (NLP para JSON)

Contexto: Transformar mensagens informais do utilizador em tarefas estruturadas para a API do ClickUp.

Input: "Preciso de um design para a home page até sexta com prioridade alta."

Tarefa: Use o Gemini com responseMimeType: "application/json" e um schema Zod para extrair:

{
  "title": string,
  "due_date": string (ISO),
  "priority": "urgent" | "high" | "normal" | "low",
  "department": "design" | "dev" | "marketing"
}
Dica: Use zodToJsonSchema para passar as definições de tipo ao modelo.
Streaming Database

3. Transcritor de Reuniões em Tempo Real

Contexto: O utilizador cola notas de uma reunião (ex: transcrição de áudio) e o sistema gera um sumário executivo.

Tarefa:

  • Implemente um stream que gera o sumário à medida que analisa o texto longo.
  • Enquanto o stream corre, exiba um indicador de "A processar pontos chave...".
  • Guarde o resultado final na tabela meeting_summaries vinculada ao project_id.
Tabela: meeting_summaries (id, project_id, original_text, summary, created_at)
Structured Output Database

4. Triage Automática de Bugs

Contexto: Analisar reportes de erros enviados por utilizadores e decidir a gravidade.

Tarefa: Crie uma função que recebe o texto do erro, usa IA para classificar e insere automaticamente na tabela tickets se a gravidade for "crítica".

Output esperado:
{
  "error_type": "UI" | "API" | "Database",
  "severity": 1 a 10,
  "fix_suggestion": "string"
}
Objetivo: Automação onde o sistema "toma decisões" baseado no JSON da IA.
Streaming Structured Output

5. Smart Planner Semanal

Contexto: O utilizador diz o que tem para fazer na semana e a IA organiza um calendário.

Input: "Tenho que entregar o logo, ir ao dentista na terça e estudar React todos os dias."

Tarefa: O modelo deve gerar um stream de texto explicando o raciocínio e, no final, um bloco JSON com a agenda.

6. Sentiment Dashboard para Gestores

Contexto: Dashboard que mostra a "saúde" da equipa com base nos comentários das tarefas.

Tarefa: Crie um script que percorre os últimos 20 comentários (mock data), envia-os num único prompt para o Gemini e recebe um JSON consolidado:

{
  "team_mood": "happy" | "stressed" | "neutral",
  "main_blocker": "string",
  "burnout_risk": boolean
}

Resultado Final Esperado

Ao terminar estes exercícios, o aluno terá um backend Express capaz de processar linguagem natural, validar dados com Zod, entregar respostas rápidas via Streaming e manter a persistência numa Base de Dados real.
